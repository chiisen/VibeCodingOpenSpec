## Why
- 目前專案尚未有正式的變更提案來規劃一頁式網站內容，缺乏可供審閱的工作範圍說明。
- 需要整理 OpenSpec 官方 README 與相關文件的重點，確保轉換成繁體中文與英文術語並存的敘事正確無誤。
- 透過提案先行定義版面區塊與素材來源，可降低後續實作時的返工風險。

## What Changes
- 建立一頁式網站的大綱，涵蓋 Hero、Why OpenSpec、Workflow、Getting Started、CLI 快速參考、範例變更資料夾與社群連結等段落。
- 彙整並本地化官方內容，產出區塊對應的文案草稿與重點項目清單，包含必要的英文關鍵字。
- 評估並列出所需靜態資產（例如既有的 `images/OpenSpec.png`）與額外素材需求，若需新增資源會在後續變更中記錄來源。

## Impact
- 為後續實作一頁式網站提供明確的資訊架構與內容基準，協助 AI 助手與人工協作者保持一致預期。
- 先行驗證內容來源與中英文用語，可避免錯譯或遺漏核心價值主張。
- 不會立即修改既有程式碼或頁面，但會促成後續實作與驗證流程的展開。
